Experiment Guide:

  • Exp 0 (Pre-coordinated Descendants): All descendants of the primary observable entity itself (e.g., all child concepts under "Creatinine in serum" - THIS is the true pre-coordinated hierarchy)
  • Exp 1 (Fixed Component): Observable entities with EXACTLY this component (no subcomponents)
  • Exp 2 (Component Descendants): Observable entities with this component OR its descendants (includes subcomponents like HbA1c for Hemoglobin)
  • Exp 3 (Fixed Component + Property): Adds specific property constraint (e.g., substance concentration)
  • Exp 4 (Fixed Component + System): Adds specimen/system constraint (e.g., blood specimen)
  • Exp 5 (Refined Query V1): Component + Universal Measurement Property (all quantitative types) + System - Most comprehensive

Why Start with Exp 0?
